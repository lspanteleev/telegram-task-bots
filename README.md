# Telegram Task Bots

Система из двух Telegram-ботов для управления задачами на разработку веб-страниц с минимальными ресурсами.

## 🎯 Описание системы

Система состоит из двух взаимосвязанных ботов:
1. **Bot 1 (Task Receiver)** — интерфейс для заказчиков, приём и отслеживание задач
2. **Bot 2 (Task Tracker)** — админ-панель для управления задачами и изменения статусов

Оба бота работают в одном процессе и делят одну SQLite базу данных.

---

## 🤖 Bot 1: Task Receiver (Приём задач)

**Назначение:** Интерфейс для заказчиков/авторов задач

### Основные команды
- `/start` — главное меню с кнопками
- `/new_task` — создать новую задачу
- `/mystatus` — посмотреть свои задачи и их статусы
- `/help` — справка

### Процесс создания задачи

Заказчик проходит через форму с пошаговым вводом:

1. **Название задачи** — короткое название (например: "Сделать страницу контактов")
2. **Описание** — подробное описание требований
3. **Дата запуска проекта** — в формате `ДД.ММ.ГГГГ` (например: `25.05.2026`)
   - На основе этой даты **автоматически** вычисляется приоритет:
     - 🔴 **Высокий** — если до запуска ≤ 7 дней
     - 🟡 **Средний** — если до запуска 8-14 дней
     - 🟢 **Низкий** — если до запуска 15+ дней
4. **Ссылка на ТЗ** — Google Docs (или другой файл с описанием требований)
   - Пример: `https://docs.google.com/document/d/1leQSF_.../edit?usp=sharing`

### Отслеживание статуса

Команда `/mystatus` показывает все задачи автора:
```
#1 - Сделать страницу контактов
Статус: ⏳ В работе
Приоритет: 🔴 Высокий (5 дней)
Дата запуска: 2026-05-25
ТЗ: [ссылка]
👤 Назначена: @username
```

**Автоматические уведомления:** Когда администратор меняет статус задачи, автор получает сообщение:
```
🔔 Обновление статуса задачи

Задача #1: Сделать страницу контактов
Статус: 🆕 Новая → ⏳ В работе
```

## 👨‍💼 Bot 2: Task Tracker (Админ-панель)

**Назначение:** Управление и отслеживание прогресса задач

### Основные команды
- `/start` — информация о боте
- `/tasks` — список всех задач
- `/status` — статистика по статусам
- `/manage` — управление задачами (интерактивные кнопки)
- `/help` — справка

### Автоматическая регистрация администратора

**Первый** пользователь, который напишет `/manage`, становится администратором:
- Его ID запоминается как `ADMIN_USER_ID`
- Только этот пользователь может управлять задачами через `/manage`
- При попытке другого пользователя: `❌ Только администратор может управлять задачами`

### Управление задачами (кнопочный интерфейс)

Команда `/manage` показывает все задачи кнопками. При нажатии на задачу показывается:

```
📌 Задача #1
Название: Сделать страницу контактов
Описание: Полная информация...
Статус: 🆕 Новая
Приоритет: 🔴 Высокий (5 дней)
Дата запуска: 2026-05-25
Описание ТЗ: [ссылка на Google Docs]
👤 Назначена: @username
Создана: 2026-05-19 14:30
```

### Кнопки для работы с задачей

| Кнопка | Действие | Результат |
|--------|----------|-----------|
| **✋ Взять задачу** | Назначить себя исполнителем | Задача привязывается к администратору, автор получает уведомление |
| **⏳ В работе** | Перевести в работу | Статус: 🔴 Новая → 🟡 В работе. Автор получает уведомление |
| **👀 На проверке** | Отправить на проверку | Статус: ⏳ В работе → 👀 На проверке. Автор видит изменение |
| **✅ Готово** | Завершить задачу | Статус: 👀 На проверке → ✅ Готово. Финальное уведомление |
| **❌ Отменить** | Отменить задачу | Статус меняется на ❌ Отменено. Информирует автора |

### Динамический расчет приоритета ⚡

**Ключевая особенность:** При каждом нажатии на кнопку приоритет **пересчитывается** в реальном времени.

Это значит, что система учитывает текущую дату каждый раз:
- **19.05.2026** (сегодня): дата запуска 25.05 → 🔴 Высокий (6 дней осталось)
- **20.05.2026** (завтра): дата запуска 25.05 → 🔴 Высокий (5 дней осталось)
- **26.05.2026** (через день после запуска): дата запуска 25.05 → 🔴 остаётся (задача просрочена)

Это позволяет видеть **актуальный приоритет** в каждый момент времени.

## 📊 Статусы задач

| Эмодзи | Статус | Значение |
|--------|--------|----------|
| 🆕 | `new` | Новая, ещё не начиналась |
| ⏳ | `in_progress` | Задача в работе |
| 👀 | `review` | На проверке у автора |
| ✅ | `done` | Завершена успешно |
| ❌ | `cancelled` | Отменена/не нужна |

## 🗄️ База данных (SQLite)

Таблица `tasks`:

| Поле | Тип | Значение |
|------|-----|----------|
| `id` | INTEGER PRIMARY KEY | Уникальный ID задачи |
| `title` | TEXT | Название задачи |
| `description` | TEXT | Подробное описание |
| `status` | TEXT | Текущий статус (new, in_progress, review, done, cancelled) |
| `created_at` | TIMESTAMP | Когда создана |
| `updated_at` | TIMESTAMP | Последнее обновление |
| `assigned_to` | INTEGER | Telegram ID администратора, взявшего задачу |
| `assigned_username` | TEXT | Юзернейм администратора |
| `creator_chat_id` | INTEGER | Telegram ID автора задачи |
| `deadline` | DATE | Дата запуска проекта (YYYY-MM-DD) |
| `tz_url` | TEXT | Ссылка на описание ТЗ (Google Docs и т.д.) |

**Важно:** Приоритет **не хранится** в БД — рассчитывается на лету на основе `deadline`.

## ⚙️ Технические детали

### Как оба бота работают в одном процессе

```python
# main.py использует asyncio.gather() для параллельной работы
await asyncio.gather(
    dp1.start_polling(bot1),  # Bot 1 слушает Telegram API
    dp2.start_polling(bot2)   # Bot 2 слушает Telegram API (одновременно)
)
```

Оба бота работают **асинхронно** и **одновременно**, но используют одну БД.

## 🚀 Установка и запуск

### 1. Создать виртуальное окружение

```powershell
python -m venv venv
venv\Scripts\Activate.ps1  # Windows
source venv/bin/activate   # Linux/Mac
```

### 2. Установить зависимости

```bash
pip install -r requirements.txt
```

### 3. Создать файл `.env`

```bash
copy .env.example .env
```

### 4. Заполнить `.env` с токенами

Получить токены в [BotFather](https://t.me/BotFather):

```
/newbot
```

Заполнить `.env`:

```
BOT1_TOKEN=7994781523:AAFBZFmtDKeGElHxf4NmGgCUoRAWTxeGarI
BOT2_TOKEN=8857780280:AAGY83LvA3VZsQCkSVamUD5buaSDA3ugWnQ
ADMIN_ID=0
TASK_RECEIVER_CHAT_ID=0
DB_PATH=tasks.db
```

**ВАЖНО:** Оставь `ADMIN_ID=0` и `TASK_RECEIVER_CHAT_ID=0` — система будет автоматически определять их при первом использовании.

### 5. Запустить

```bash
python main.py
```

## 🐛 Решение проблем

### Бот не отвечает

```bash
# Проверь токены в .env
echo $BOT1_TOKEN

# Перезапусти бота
python main.py
```

### База данных повреждена

```bash

# Убедись, что виртуальное окружение активировано
venv\Scripts\Activate.ps1  # Windows
source venv/bin/activate   # Linux/Mac

# Переустанови зависимости
pip install -r requirements.txt
```
